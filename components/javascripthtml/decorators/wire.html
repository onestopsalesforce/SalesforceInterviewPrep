Understanding the @wire Decorator in Salesforce Lightning Web Components

The @wire decorator in Lightning Web Components (LWC) is primarily used in three contexts:

  • Calling Apex methods to fetch custom server-side data
  • Leveraging UI API wire adapters to retrieve record data, object metadata, and picklist values
  • Accessing user and navigation data using built-in services like getUserInfo and CurrentPageReference

This blog explores all these use cases with examples to help you understand how to use @wire effectively.

1. Importing @wire

Before using @wire, you must import it from the LWC module: 

import { LightningElement, wire } from 'lwc';

2. UI API Modules (lightning/ui*)

These built-in wire adapters allow you to retrieve Salesforce metadata and record-level data without writing Apex code.

  • getRecord – Retrieves specific fields from a record.
  • getObjectInfo – Fetches metadata about an object (label, field details, etc.).
  • getPicklistValues – Retrieves picklist options for a given record type and field.
  • getListInfo – Retrieves metadata about a list view.
  • getRelatedListInfo – Fetches metadata about related lists.

Example: Fetching a record’s name and phone:


  @wire(getRecord, {
    recordId: '$recordId',
    fields: ['Account.Name', 'Account.Phone']
  }) account;

3. User Info via lightning/user

This module includes a wire adapter to get information about the logged-in user.

  • getUserInfo – Returns the current user's ID, name, profile ID, and more.

Example: Fetching the user’s name:


  @wire(getUserInfo) user;

4. Navigation Context via lightning/navigation

  • CurrentPageReference – Retrieves the current page's context including URL state and query parameters.
  • getNavItems – Retrieves navigation menu items for custom apps and tabs.

Example: Accessing a URL query parameter like c__filter:


  @wire(CurrentPageReference) pageRef;
  
  get filter() {
    return this.pageRef?.state?.c__filter;
  }

5. Custom Apex Methods

You can wire custom Apex methods only if they are annotated with @AuraEnabled(cacheable=true). This enables reactive, cacheable server-side data fetching.

Important: If cacheable=true is not specified, the system will throw a runtime error when the method is used with @wire. Only cacheable Apex methods can be used declaratively with the @wire decorator.

Example: Fetching accounts filtered by industry:


  // Apex
  @AuraEnabled(cacheable=true)
  public static List<Account> getAccounts(String industry) {
    return [SELECT Id, Name FROM Account WHERE Industry = :industry];
  }
  
  // JavaScript
  @wire(getAccounts, { industry: '$industry' }) accounts;

In this example, any change in the industry property will automatically trigger a new call to the Apex method, and the component will re-render with the updated data.
