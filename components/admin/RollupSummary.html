Understanding Roll-Up Summary Fields in Salesforce

In Salesforce, Roll-Up Summary fields are used to aggregate values from related child records to summarize information on the parent record. These fields allow you to calculate sums, counts, minimum, or maximum values based on the child records in a master-detail relationship. Here's what you need to know about Roll-Up Summary fields:

Master-Detail Relationships

Roll-Up Summary fields can only be created in **Master-Detail relationships**. In this relationship type, the parent record controls the behavior of the related child records. This means that Roll-Up Summary fields can aggregate data from the child objects to the parent object.

However, you cannot create Roll-Up Summary fields in **Lookup relationships** unless you use third-party tools like Declarative Lookup Rollup Summary (DLRS) or custom solutions like Apex triggers or **Flows**.

Creating Roll-Up Summary Fields

You can create Roll-Up Summary fields based on **Formula fields** or **Custom fields** in the related child objects. This gives you flexibility to aggregate data from various field types on the child records.

Handling Large Data Volumes

When dealing with large data volumes, it's important to consider performance. Salesforce Roll-Up Summary fields can be slow when processing large numbers of child records, especially if there are more than **10,000** child records involved in one roll-up calculation.

To handle large datasets, it's advisable to use tools like **Declarative Lookup Rollup Summary**, **Apex triggers**, or **Scheduled batch jobs** to periodically recalculate and update these fields efficiently.

Limitations in Roll-Up Summary Fields

  • Roll-Up Summary fields **do not automatically recalculate** when child records (like Leads, Contacts, or Campaign Members) are deleted.
  • Salesforce provides an option to manually update Roll-Up Summary fields called **"Force a mass recalculation of this field"**. This option can be enabled while editing the Roll-Up Summary field configuration in Salesforce Setup.

To enable this option, follow these steps:

  1. Navigate to Setup > Object Manager.
  2. Select the object that contains the Roll-Up Summary field.
  3. Click on the Roll-Up Summary field you want to update.
  4. In the Roll-Up Summary Options, select **Force a mass recalculation of this field**.

Limits on Roll-Up Summary Fields

Salesforce places some limits on Roll-Up Summary fields to optimize performance:

  • You can create up to **25 Roll-Up Summary fields** per object. This limit includes both standard and custom Roll-Up Summary fields.
  • Supported aggregation functions include:
    • Sum
    • Min
    • Max
    • Count
  • Roll-Up Summary fields support the following data types for aggregation:
    • Number
    • Currency
    • Percent
    • Formula (if it returns a supported data type)
