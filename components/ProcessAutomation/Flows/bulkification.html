Understanding Salesforce Flow Bulkification

Salesforce automatically bulkifies Flow transactions to optimize processing and stay within platform limits. However, the efficiency depends on the type of Flow and how it's executed.

1. What is Flow Bulkification?

✔️ Bulkification ensures that multiple records are processed in a single transaction, reducing SOQL queries and DML operations.

2. How Different Flow Types Handle Bulkification

A. Record-Triggered Flows

  • ✔️ Processes records in batches (up to 200 at a time).
  • ✔️ Avoids hitting SOQL and DML limits.
  • ⚡ Requires careful handling of loops and collections.

B. Scheduled Flows

  • ✔️ Runs in bulk like Batch Apex.
  • ✔️ Processes records efficiently at a scheduled time.

C. Autolaunched Flows

  • ✔️ Called by Apex or Process Builder.
  • ⚡ Needs explicit bulk handling when triggered via Apex.

D. Screen Flows

  • ❌ Processes one record at a time.
  • ❌ Not inherently bulkified.

3. Common Mistakes That Break Bulkification

  • ❌ Placing DML statements inside loops.
  • ❌ Querying records multiple times inside a loop.
  • ❌ Not using collection variables.

4. Best Practices for Bulkified Flows

  • ✔️ Use collection variables for record processing.
  • ✔️ Minimize SOQL queries and avoid unnecessary loops.
  • ✔️ Use bulk operations like "Update Records" instead of single-record updates.
  • ✔️ Test with bulk data using Data Loader.

5. Example: Bulkifying a Record-Triggered Flow

// ❌ Bad Approach (Non-Bulkified)
For each record in trigger:
   Get related records
   Update one record at a time

This leads to multiple SOQL queries and DML statements.

// ✔️ Good Approach (Bulkified)
Get all related records in one SOQL query
Process records in loop (without querying again)
Use "Update Records" to update all at once

This reduces the number of SOQL queries and DML operations.

6. Governor Limits and Transactions

⚡ If any interview hits a governor limit, all the interviews in the transaction fail. Any operations that the interviews performed in the transaction are rolled back, and the transaction doesn’t try to perform the operations again. Any operations that access external data aren’t rolled back.

Conclusion

✔️ Salesforce bulkifies Flows automatically, but improper design (e.g., DML inside loops) can lead to governor limits. Following best practices ensures optimal performance.
