Approval Process in Flows - Quick Refresher

This guide serves as a refresher for experienced developers preparing for interviews. It covers key aspects of the approval process in Salesforce Flows.

Approval Process Components

Entry Criteria

  • Defines the conditions under which a record enters the approval process.

Approver Field

  • Selection #1 - Approver Field:
    • None
    • Manager
    • Hierarchical Relationship
    • If left blank, cannot be changed unless an existing Custom Hierarchical Relationship is present.
  • Selection #2 - Use Approver Field of Lead Owner:
    • Determines whether the criteria should be based on the Record Owner rather than the submitted user.
  • Selection #3 - Record can be updated by:
    • Administrator only
    • Administrator and Approver

Approval Assignment Template (Optional)

  • Predefined template for assigning approvals.

Approval Screen

  • Approver - Record Fields that can be viewed
  • Display Approval History
  • Availability:
    • Only on Salesforce Application
    • Mobile Device too

Initial Submitter

  • Defines who can submit records for approval:
    • Creator
    • Owner
    • Public Groups
    • Role
    • Role and Internal Sub-Ordinates
    • User
  • Determines whether the submitted user can recall the approval request.

Approval Process Setup

  • Initial Submission Actions
  • Approval Steps
  • Final Approval Actions
  • Final Rejection Actions
  • Recall Actions

Approval Situations

  • Auto Approval - Approval is granted automatically if conditions are met.
  • Delegation Approval - Approval delegated to another user.
  • One Level Approval:
    • Single Member Approval
    • Multi-Member - Unanimous approval required
    • Multi-Member - Any member of the group can approve
  • Multi-Level Approval:
    • Manager Approval required only if certain conditions are met.
    • Manager and Senior Manager Approval required.
    • Manager approval not required, but Senior Manager approval required.

Additional Considerations

  • Rejection Handling:
    • Define whether a rejected record can be resubmitted.
    • Set up notifications for rejections.
  • Escalation Rules: Approvals can be escalated if no action is taken within a specified timeframe.
  • Parallel Approvals: Multiple approvers can review a request simultaneously.
  • Notifications: Email alerts and in-app notifications for approvals and rejections.
